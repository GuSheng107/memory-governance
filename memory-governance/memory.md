# Global Memory

## Stable Rules

- Read memory before coding, debugging, analysis, test, or refactor work.
- When project memory exists, it overrides global memory inside that project.
- Never create `.ai-memory/` without explicit user confirmation.
- Keep the memory system file-based only; do not use a database or hidden state store.
- Write fresh lessons to `corrections.md` first and promote only stable rules to `memory.md`.
